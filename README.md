# QA Artifacts & Work Samples — Mohamed Ali Atallah

A curated set of **QA deliverables** that demonstrate how I plan, document, automate, and report testing as a Senior QA Engineer (Manual · Automation · API · Performance).

> ℹ️ These are **illustrative work samples** built around a generic demo e-commerce / rental web application. They reflect the structure, depth, and methodology I use on real projects — without exposing any confidential client data.

## Contents

| Folder | What's inside |
|--------|---------------|
| [`bug-reports/`](./bug-reports) | Structured defect reports (severity, priority, environment, steps, expected vs. actual, evidence). |
| [`test-plans/`](./test-plans) | An IEEE-829-style test plan: scope, strategy, entry/exit criteria, risks, traceability. |
| [`api-testing/`](./api-testing) | A Postman collection + environment with positive, negative, and schema-validation tests. |
| [`performance-k6/`](./performance-k6) | A Grafana k6 load/stress script with thresholds and staged ramping. |
| [`ci-cd-jenkins/`](./ci-cd-jenkins) | A declarative `Jenkinsfile` that runs API + UI regression on every build and publishes reports. |
